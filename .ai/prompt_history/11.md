## Plan: Security Governance Artifacts

Projeye Spring Authorization Server tabanlı, operation-claim yetkilendirmeli, hem gateway hem servis katmanında token doğrulayan bir güvenlik mimarisi eklenecek. Bu plan yalnızca **governance çıktılarını** (AGENTS.MD, DECISIONS.MD, agent dosyası, prompt dosyaları) kapsar; servis implementasyonu ayrı bir görevdir.

---

### Kararlar Özeti (Governance'a İşlenecek)

| # | Karar | Değer |
|---|---|---|
| D007 | Auth Framework | Spring Authorization Server (yeni `auth-service` modülü) |
| D008 | Token Doğrulama | Gateway + her servis (OAuth2 Resource Server, defense-in-depth) |
| D009 | Yetkilendirme Modeli | Operation-Claim Based (`Product.Create`, `Order.Cancel` vb.) |
| D010 | Public Endpointler | `GET /api/v1/products` ve `GET /api/v1/products/{id}` |
| D011 | auth-service User Store | H2 In-Memory |
| D012 | Claim Kümesi | 7 claim — Product.Read/Create/Update/Delete, Order.Read/Create/Cancel |

---

### Adımlar

**1. DECISIONS.MD — D007–D012 eklenir**
- Her karar için `ID`, `Başlık`, `Karar`, `Gerekçe`, `Tarih` formatı korunur
- Claim → endpoint eşlemesi tablo olarak eklenir
- Public endpoint listesi net biçimde belgelenir

**2. AGENTS.MD — Yeni "5. Security" bölümü eklenir**
- Tüm endpointlerin OpenAPI spec'inde `securitySchemes` ve `security` alanı zorunlu hale getirilir (public olanlar `security: []` ile açıkça belgelenir)
- "Operation-Claim bazlı yetkilendirme" kuralı: claim → endpoint eşleşmesi DECISIONS.MD'den alınır, uydurulamaz
- auth-service modülünün yetki alanı tanımlanır (token ihracı, kullanıcı/claim yönetimi)
- Gateway ve her servis için OAuth2 Resource Server yapılandırması zorunlu tutulur

**3. `.github/agents/security-agent.agent.md` — Yeni agent dosyası**

Senior Java Dev agent yeterli **değil** çünkü:
- Spring Security filter chain yapılandırma kuralları yok
- `SecurityFilterChain`, `@PreAuthorize`, `jwt()` converters için kural yok
- Hangi claim kümesinin kullanıldığına dair referans yok
- `auth-service` vs resource server ayrımı yok

Yeni agent şunları içerecek:
- Security-specific discovery adımları (SecurityConfig, SecurityFilterChain, `application.yml` `spring.security.oauth2.*` bloklarını okuma)
- Claim bazlı `@PreAuthorize("hasAuthority('Product.Create')")` pattern kuralı
- `jwk-set-uri` / `issuer-uri` config kuralı — hardcode yasak
- Public endpoint değişikliği için DECISIONS.MD güncellemesi zorunluluğu
- OpenAPI spec güncelleme zorunluluğu (her security değişikliğinde)

**4. `.github/prompts/protect-endpoint.prompt.md` — Yeni prompt dosyası**

Belirli bir endpoint'i doğru claim ile korumaya almak için yeniden kullanılabilir prompt:
- Hangi servis, hangi endpoint, hangi HTTP metodu → hangi claim?
- OpenAPI spec'i güncelle (`security` alanı ekle)
- Controller'da `@PreAuthorize` ekle
- Security config'de kuralı doğrula
- Test adımı: geçerli claim içeren JWT vs içermeyen JWT davranışı

**5. `.github/prompts/add-claim.prompt.md` — Yeni prompt dosyası**

Sisteme yeni bir claim eklemek için yeniden kullanılabilir prompt:
- DECISIONS.MD D012'yi güncelle (claim kümesini genişlet)
- auth-service'te claim entity'si / seed data ekle
- İlgili endpoint'i koru (protect-endpoint prompt'unu çağırır)
- OpenAPI spec güncelle
