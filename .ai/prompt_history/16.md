**Bölüm 2 — Katman Tanımları ve Sorumlulukları**

| Katman | Paket | Sorumluluk |
|---|---|---|
| **Domain** | `domain.model` | Entity, Value Object, Enum, domain exception |
| **Domain** | `domain.port.in` | Inbound port interface'leri (use case tanımları) |
| **Domain** | `domain.port.out` | Outbound port interface'leri (repository, external service contract) |
| **Application** | `application.service` | Use case implementasyonları, orchestration, transaction yönetimi |
| **Adapter In** | `adapter.in.web` | REST controller'lar, request/response DTO'lar, exception handler |
| **Adapter In** | `adapter.in.web.dto.request` | Request DTO'ları |
| **Adapter In** | `adapter.in.web.dto.response` | Response DTO'ları |
| **Adapter In** | `adapter.in.web.mapper` | Web katmanı mapper'ları (DTO ↔ Domain) |
| **Adapter Out** | `adapter.out.persistence` | JPA entity, Spring Data repository, persistence mapper |
| **Adapter Out** | `adapter.out.feign` | Feign client'lar ve ilgili DTO'lar |
| **Config** | `config` | Spring configuration sınıfları (Security, Feign vb.) |

**Bölüm 3 — Bağımlılık Kuralları**
- Domain katmanı hiçbir dış katmana bağımlı olamaz (framework import yasak)
- Application katmanı yalnızca domain katmanına bağımlıdır
- Adapter katmanları application ve domain katmanlarına bağımlı olabilir
- Adapter'lar birbirine bağımlı olamaz
- Dependency direction diyagramı (metin tabanlı)

**Bölüm 4 — Örnek Paket Ağacı**
- product-service için tam paket ağacı örneği:
  ```

  │   ├── port

  │   │   │   └── ProductService.java (use case interface)
  │   │   └── out
  │   │       └── ProductRepository.java (port interface)
  ├── application
  │   └── service

  ```

- gateway-server: Routing + security config, business logic yok — flat yapı

### Dosya 2: `.ai/architecture/CODING_STANDARDS.MD`

İçerik planı (~250-300 satır, her kural doğru/yanlış örnekle):

**Bölüm 1 — Genel İsimlendirme Kuralları**
- Sınıflar: `PascalCase` (ör: `ProductService`, `OrderController`)
- Metotlar ve değişkenler: `camelCase` (ör: `findById`, `orderStatus`)
- Sabitler: `UPPER_SNAKE_CASE` (ör: `MAX_PAGE_SIZE`)
- Paketler: `lowercase`, tek kelime tercih edilir (ör: `domain.model`, `adapter.in.web`)
- Boolean değişkenler: `is`/`has`/`can` prefix'i ile (ör: `isActive`, `hasStock`)

**Bölüm 2 — Sınıf İsimlendirme Konvansiyonları**

| Tür | Suffix/Pattern | Örnek |
|---|---|---|
| REST Controller | `*Controller` | `ProductController` |
| Use Case Interface (Inbound Port) | `*Service` | `ProductService` |
| Use Case Impl | `*ServiceImpl` | `ProductServiceImpl` |
| Outbound Port Interface | `*Repository` / `*Client` | `ProductRepository`, `ProductClient` |
| JPA Repository (Spring Data) | `*JpaRepository` | `ProductJpaRepository` |
| Persistence Adapter | `*PersistenceAdapter` | `ProductPersistenceAdapter` |
| Feign Adapter | `*FeignAdapter` | `ProductFeignAdapter` |
| JPA Entity | `*JpaEntity` | `ProductJpaEntity` |
| Domain Model | Suffix yok | `Product`, `Order` |
| Request DTO | `*Request` | `ProductCreateRequest` |
| Response DTO | `*Response` | `ProductResponse` |
| Mapper | `*Mapper` / `*WebMapper` / `*PersistenceMapper` | `ProductWebMapper` |
| Exception | `*Exception` | `ProductNotFoundException` |
| Config | `*Config` | `SecurityConfig` |
| Enum | Suffix yok, PascalCase | `OrderStatus` |

**Bölüm 3 — DTO Kuralları**
- Request DTO: `adapter.in.web.dto.request` paketinde
- Response DTO: `adapter.in.web.dto.response` paketinde
- Jakarta Bean Validation request DTO'larda uygulanır
- DTO'lar `record` yerine class olarak tanımlanır (mevcut konvansiyona uyumlu)
- DTO'larda Lombok kullanılmaz (mevcut kurala uyumlu)

**Bölüm 4 — Exception Handling Standardı**
- Tüm servislerde `@RestControllerAdvice` ile `GlobalExceptionHandler` kullanılır
- Handler `adapter.in.web` paketinde yer alır
- Tüm hata yanıtları `ErrorResponse` DTO kullanır (`code`, `message`, `details` alanları)
- HTTP status `ResponseEntity<ErrorResponse>` ile açıkça döndürülür (`@ResponseStatus` annotation kullanılmaz — tutarsızlık giderilir)
- Domain exception'lar `domain.model` veya `domain.exception` paketinde tanımlanır

**Bölüm 5 — Dependency Injection**
- Constructor injection zorunlu
- `@Autowired` annotation kullanılmaz
- Lombok (`@RequiredArgsConstructor`) kullanılmaz
- `final` field'lar ile immutability sağlanır

**Bölüm 6 — Service Katmanı**
- Inbound port: interface olarak `domain.port.in` paketinde
- Implementation: `application.service` paketinde, `*ServiceImpl` ismiyle
- `@Service` annotation implementation sınıfına eklenir
- `@Transactional` application service seviyesinde uygulanır

**Bölüm 7 — Controller Katmanı**
- `@RestController` ve `@RequestMapping` sınıf seviyesinde
- `@Validated` sınıf seviyesinde
- Metot dönüş tipi: `ResponseEntity<T>`
- Security: `@PreAuthorize("hasAuthority('...')")` DECISIONS.MD D009/D012 ile uyumlu
- Swagger: `@Operation`, `@ApiResponse` annotation'ları ile dokümante edilir

**Bölüm 8 — Entity ve Domain Model**
- Domain model: framework-free POJO, JPA annotation yasak
- JPA entity: `adapter.out.persistence` paketinde, `*JpaEntity` suffix ile
- Primary key: `UUID`, `@GeneratedValue(strategy = GenerationType.UUID)`
- Manuel getter/setter (Lombok yok)

**Bölüm 9 — Mapper**
- Hand-written `@Component` mapper (MapStruct yok — mevcut kurala uyumlu)
- İki tür mapper:
  - `*WebMapper`: DTO ↔ Domain (adapter.in.web.mapper)
  - `*PersistenceMapper`: Domain ↔ JPA Entity (adapter.out.persistence)
- Static metot yerine instance metot tercih edilir

**Bölüm 10 — Kod Formatı**
- Satır uzunluğu: maksimum 120 karakter
- Import sırası: `java.*`, `jakarta.*`, `org.*`, `com.*` — wildcard import yasak
- Boş satır: sınıf seviyesi üyeler arasında tek boş satır
- Parantez stili: K&R (açılış parantezi aynı satırda)

**Bölüm 11 — Test İsimlendirme**
- Test sınıfı: `*Test` suffix (ör: `ProductServiceImplTest`)
- Test metodu: `should_DoSomething_When_Condition` pattern
- Test paketi: ana kaynak ile aynı paket yapısı
